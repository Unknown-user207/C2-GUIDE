# Lessons Learned — ShadowC2

## 1. Start Simple
- Don't over-engineer from day one
- Build a working core first, then add features

## 2. Test on Real Devices
- Emulators are good, but real devices reveal network and permission issues

## 3. Log Everything
- Logs saved to `shadow_journal.md` and `guide_memory.db`
- Helps debug and track progress

## 4. Keep the Guide Fed
- The more knowledge you add to `knowledge/`, the smarter the Guide becomes

## 5. Silence is Power
- The mute nature of the Guide is not a limitation — it's a focus amplifier
